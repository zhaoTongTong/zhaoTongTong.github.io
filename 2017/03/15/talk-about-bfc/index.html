BFC那些事

发表于 2017-03-15

前言

　　弄清BFC之前首先需要了解FC的概念，FC（formatting context）直译就是格式化上下文，它的本质是页面中的一块渲染区域，其中有一套渲染规则来规定里面的元素应该怎么渲染。现在常见的FC有BFC（block formatting context）以及IFC（inline formatting context）。BFC是块级盒子参与的渲染区域，IFC是行级盒子参与的渲染区域。
　　说到这里，再来提一下这个块级盒子的概念，块级盒子就是display 属性为 block, list-item, table 的元素，会生成 block-level box，参与BFC渲染。下面要给出BFC的一个定义，属于个人理解。

BFC的定义

　　BFC是页面内一个独立的渲染区域，只有块级盒子能参与其中，并且它拥有一套渲染规则，规定了内部的块级盒子如何布局。同时，BFC内外互不影响。

BFC的渲染规则

  1. 内部的Box会在垂直方向，一个接一个地放置。
  2. Box垂直方向的距离由margin决定。属于同一个BFC的两个相邻Box的margin会发生重叠
  3. 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。
  4. BFC的区域不会与float box重叠。
  5. BFC就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。
  6. 计算BFC的高度时，浮动元素也参与计算

哪些元素会触发BFC？

  • 根元素
  • float属性不为none
  • position为absolute或fixed
  • display为inline-block, table-cell, table-caption, flex, inline-flex
  • overflow不为visible

BFC的作用

　　说起BFC的作用，也许很容易和清除浮动混淆，实际上清除浮动只是它的作用之一。概括的说它主要有3个用途：

  • 用于布局
  • 用于清除浮动
  • 用于避免margin重叠
