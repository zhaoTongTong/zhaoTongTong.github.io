passport调研（百度、京东）

发表于 2017-10-31

passport初步调研

百度passport

登陆流程：（有弹框登陆，使用 jsonp 跨域）
  1. 进入百度页面时自动获取BAIDUID（服务器种 cookie 的方式），若没有该ID则无法获取token和codestr；
  2. 登陆前获取 token 和 codestr（一个jsonp请求 https://passport.baidu.com/v2/api/?getapi&tpl=ik&apiver=v3&tt=1510712838411&class=login）；
  3. 用 codestr 获取验证码图片 https://passport.baidu.com/cgi-bin/genimage?+ codestr ；
  4. 密码输入框获得焦点时，发请求获得一个 rsakey 用于密码加密，该值每次请求都会变化
  5. 用 token、codestr、verifycode（验证码）、username、passport（加密后的）、rsakey 等数据发送POST请求（猜测是CORS处理跨域），登陆成功接收到一个html；
  6. 该 html 内容如下，将浏览器 location.href 定向到另一个 html 页面，并附带所有登陆参数；
  7. 该 html 内容如下，将登陆参数用 config 对象保存，并作为参数调用 callback ，跳回登陆前refer页面；
手机验证码发送流程：

  1. 点击发送验证码按钮后，会将 token、gid（前端随机算法生成的id）、username、时间戳等信息 jsonp 请求后端，若不是当日第一次请求则需填写验证码，即返回 vcodesign、vcodestr 用于获取验证码；

  2. 填写验证码后即会发送短信验证码；此时若使用该 jsonp 请求的 url 用浏览器访问会返回“操作过于频繁”错误，其按照 gid 为标记做了计时60s内无法重复发送验证码，而60s后再次访问该 url 报“验证码已失效”错误， 即验证码的有效期为60s。所以一个发送短信的 jsonp 请求 url 是一次性的，必须60s后更换验证码才能重复发送短信。

京东passport

登陆流程：(无弹框登陆，均为 passport 页面登陆，无需 jsonp 跨域）
  1. 进入登陆页面时，访问 https://passport.jd.com/new/login.aspx? 接口，服务器将返回一个带有下图所示 input 表单的 html 页面，即登陆页，input 里的数据都是登陆所需的。其中同一账号同一设备多次登陆时，仅uuid发生变化。
  2. 输入用户名密码后点击登陆即发送一个POST请求，其中 eid 及 fp 字段为前端计算得到，其他数据均为服务器用 input 带过来的，请求数据截图如下所示，请求成功后返回一个跳转地址 url ；

公司passport改善方案

  1. 针对 jsonp 请求进行 refer 过滤， 非白名单内的 refer 拦截或者用 CORS 替换 jsonp 进行跨域处理；
  2. 做一次性token，例如输入框每次聚焦从服务器获取新的token做校验；
  3. 针对短信验证码的问题，可以效仿百度做计时，并给验证码一个有效时长，避免现在可以一直发送的现象；
